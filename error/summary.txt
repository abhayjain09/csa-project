## Root causes, ranked by note frequency

**1. One document mapped to every question (biggest single driver — Ball).** Every Ball FP note is identical: *"Corporate Income Tax Report has been downloaded and mapped in S3 for all the questions."* Summary row: *"Facing the issue mainly mapping of the same document to multiple questions."* This is a downstream retrieval/mapping bug (`app.py`, not yet reviewed) — one stored doc is answering every question class because retrieval isn't filtering by `doc_class`. Accounts for essentially all 18 Ball FPs.

**2. Wrong-company mapping (Edwards — total failure).** Every FP: *"Mapped to a wrong company."* 0/23 correct. "Edwards" is a common/collision name; the company-identity gate isn't catching cross-company contamination here.

**3. Proxy statement grabs the wrong SEC form (Intuit, DaVita, Cisco — all confirmed by name).** Intuit: *"should be getting DEF... instead downloading the Additional DEF."* DaVita: *"downloading 3 pages additional one but not the main document."* Cisco: *"downloaded in additional Proxy not the original one."* This is the DEFA14A-vs-DEF14A bug — three independent companies confirm the exact same failure mode. (Already patched in `report_specs.py`/`agent.py` in this session; not yet re-tested against this sheet.)

**4. Freeport-McMoRan — same one-doc-to-many-questions bug as Ball**, just phrased differently: *"wrong document has been mapped in the S3 document for all the document"* repeated across Code of Conduct, Anti-Corruption, Insider Trading, Discrimination, H&S, Human Rights. Same root cause as #1, different company.

**5. Recency misses.** CBRE Sustainability Report: *"Latest document was not downloaded."* Freeport Sustainability: *"Available but downloaded the random documents."*

**6. Ball's Sustainability/Annual Report is a merged document.** *"Company has both AR and SR in a single document but some document has been downloaded"* — a structural edge case (combined filing) the class-verifier isn't handling.

**7. Intuit's high accuracy is largely TN, not real capture.** 10 of its 17 TNs share one note: *"policy information is on the website"* — i.e., the agent correctly stores nothing because there's no PDF, but the actual policy content exists as HTML and isn't being captured. Inflates the accuracy number without reflecting real coverage.
