Home / Architecture

Architecture

A multi-agent system on Amazon Bedrock AgentCore. Agents run on AgentCore Runtime and coordinate over A2A; tools (web search, S3, DynamoDB, the knowledge base, the browser) are reached through AgentCore Gateway.

AgentCore Runtime agents AWS data stores Evaluation Output

Key decisions

Direct RAG, or store files in S3 first?

Bedrock Knowledge Bases support three ingestion paths: a Web Crawler data source that points straight at official URLs (true "direct RAG", no file in your bucket); a custom data source that ingests document bytes inline via the KnowledgeBaseDocuments API (no S3); or an S3 data source. Recommendation: keep S3 as the system of record and ingest from it — the frozen original gives you an audit trail, reproducible re-chunking when you tune the pipeline, and a stable target for the provenance record. Use the Web Crawler path as a fallback for HTML-only pages with no downloadable file.

Where the web query is saved

DynamoDB web_queries table — structured, queryable run config the search agent reads directly. A write fires an SQS message (or EventBridge rule) that starts the pipeline. S3 is reserved for report blobs; Parameter Store / AppConfig are viable but keep query, status, and results split across services.

Quality gate on every answer

The Evaluator uses AgentCore Evaluations to score each answer on retrieval, groundedness, and completeness. Anything below threshold is routed back to the Verifier agent, which re-checks the live official site with the Browser Tool before the answer is allowed into the final report.
